## 1. Введение и выбор технологии
Финальный этап инфраструктурной разработки проекта (Раздел 6.5) потребовал подключения нашего поискового движка `ATSRanker` (описанного в Разделе 6.4) к внешнему миру.
В качестве базовой технологии был выбран фреймворк **FastAPI**. 
Его преимущества для машинного обучения очевидны: 
- Автоматическая валидация данных (с помощью схемы `Pydantic`).
- Экстремально высокая пропускная способность за счет асинхронного ASGI ядра (`uvicorn`).
- Встроенная документация Swagger UI для легкого тестирования ML-модулей не-техническим HR-персоналом.

## 2. Реализованные интерфейсы
Были запрограммированы следующие ключевые маршруты связи (Endpoints):

*   **`POST /index/candidate`**:
    Принимает профиль кандидата (строгий контракт `CandidatePayload`) и "пробрасывает" его во внутренний кэш или БД системы. Таким образом симулируется поток входящих откликов.
*   **`POST /search/rank`**:
    Сердце системы. Принимает данные вакансии (`SearchPayload`, который включает запрошенные хард-фильтры и значение `top_K`). Вызывает метод `ATSRanker.search`, который делает фильтрацию, векторизует (Bi-Encoder Dense Retrieval) и отдает готовый JSON ответ.
*   **`POST /score`**:
    Минимальный service-to-service маршрут для интеграции с Reqcore. Принимает пару `{ vacancy_text, candidate_text }`, вычисляет cosine similarity и возвращает числовой `match_score` для score-only сценариев.

## 3. Мониторинг и Метаинформация (Middleware Telemetry)
Для обеспечения Observability (наблюдаемости), требуемой при тестировании научных ML моделей в production:
- Написано специальное HTTP-Middleware (`add_telemetry_headers`).
- Сервер "перехватывает" время начала запроса и время его завершения. 
- На выходе в HTTP Response Headers добавляются:
  *   `X-Processing-Time-Ms` (физическая задержка на обработку нейросети).
  *   `X-Model-Version` (basename от `MODEL_PATH`), чтобы при A/B тестировании ATS система точно знала, какой артефакт модели обслужил запрос.

Для запуска сервиса требуется явная runtime-переменная окружения `MODEL_PATH`; скрытый fallback на локальный путь разработчика не используется.

## Практические выводы для интеграции
Проектирование и запуск `11_api_server.py` переводит алгоритмическую плоскость диссертации в воспроизводимый HTTP-контур для интеграции с ATS. В текущем виде это локальный FastAPI-микросервис с in-memory индексом кандидатов и минимальным score-only маршрутом для Reqcore. Утверждения о промышленном масштабировании или миллионах запросов в данном отчёте не фиксируются без отдельного нагрузочного измерения.
