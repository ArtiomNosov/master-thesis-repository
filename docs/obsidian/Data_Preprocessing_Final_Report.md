# Итоговый отчет: Препроцессинг пайплайн (Раздел 6.2)

#thesis #report #preprocessing #ats #feature_engineering

## 1. Введение
В рамках этапа разработки препроцессинга (Раздел 6.2) был сконструирован конвейер извлечения семантических признаков и нормализации текста для корпусов резюме и вакансий. Аналогично предыдущему этапу, обработка велась с соблюдением требований **Zero-Dependency Batch Processing** для защиты от превышения лимитов оперативной памяти (менее 1 ГБ). 

## 2. Нормализация текста (05_text_normalization)
Текстовые поля (названия профессий, описания, обязанности) подверглись следующей обработке:
- Приведение к нижнему регистру (lowercasing).
- Удаление пунктуации с сохранением значимых ИТ-символов (например, знаки `+`, `#` для C++, C#).
- Вырезание шаблонных HR-блоков ("Обязанности:", "Условия:", "Описание вакансии") с помощью регулярных выражений для снижения шума в эмбеддингах.

## 3. Извлечение сущностей (06_entity_extraction)
Разработан скрипт поиска по словарным базам для извлечения ключевых метаданных:
- **Уровень квалификации (Seniority markers):** Успешно извлекаются категории `Intern/Junior`, `Middle`, `Senior`, `Lead`, что позволяет моделям явно разделять кандидатов при матчинге.
- **Ключевые навыки (Skills & Technologies):** Из плоского текста вытягиваются факты владения базовыми технологическими стеками (Python, SQL, React, Git, Docker, Kubernetes и др.), и формируются в виде JSON-списков.

## 4. Унификация формата для моделей (07_feature_formatting)
Наиболее важным этапом стала сборка извлеченных признаков обратно в потоковый формализованный текст для Bi-Encoder моделей (например, `sbert` или `e5`).
Был сформирован единый текстовый вид (`model_input`), который объединяет контекст:
`Role: [роль] | Seniority: [уровень] | Skills: [список навыков] | Text: [очищенное описание]`

## Практические выводы для диссертации
1. Очистка от шаблонных HR-фраз критически снижает вероятность того, что векторная модель найдет "ложное сходство" между двумя вакансиями просто из-за фразы "Мы предлагаем оформление по ТК РФ".
2. Явное выделение `Seniority` и `Skills` в префиксе текста служит сильным сигналом (hard prompt) для Attention-механизмов больших языковых моделей (LLM / Transformer), улучшая способность разделять синьоров и джуниоров.
3. Потоковая архитектура конвейера препроцессинга подтвердила свою надежность: обработка тысяч строк отнимает минимальное количество RAM и независима от установки громоздких ML-фреймворков. Датасет полностью готов к этапу векторизации (эмбеддингов).
