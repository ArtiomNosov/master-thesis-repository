## Критерий 1: Работающий программный прототип или модуль

**✅ ВЫПОЛНЕН**

Разработан полноценный ATS AI Engine, состоящий из **17 взаимосвязанных Python-скриптов** в папке `experiments/scripts/`:
- `05_text_normalization.py` — нормализация данных
- `06_entity_extraction.py` — извлечение сущностей
- `07_feature_formatting.py` — унифицированный формат признаков
- `08_model_dataset.py` + `09_train_biencoder.py` — обучение модели
- `10_ranking_module.py` — ядро ранжирования (`ATSRanker`)
- `11_api_server.py` — FastAPI микросервис
- `12_`–`17_` — экспериментальные бенчмарки

Все скрипты исполняются без ошибок на Python 3.11 (проверено многократным запуском в ходе разработки).

## Критерий 5: Сравнение с baseline-подходами выполнено

**✅ ВЫПОЛНЕН**

Скрипт `12_baseline_comparison.py` зафиксировал протокол сравнения ровно трех подходов:

| Подход | Роль в эксперименте |
|---|---|
| BM25 lexical ranking baseline | Классическая лексическая нижняя граница без обучения |
| Fine-tuned Cross-Encoder `cointegrated/rubert-tiny2` | Верхняя граница качества при дорогом pairwise scoring |
| Fine-tuned Bi-Encoder `experiments/models/bi_encoder_rubert_tiny2` | Масштабируемый proposed approach для ATS |

Результаты и выводы: `docs/obsidian/Baseline_Comparison_Report.md`.

## Критерий 6: Интеграция с ATS или тестовым контуром завершена

**✅ ВЫПОЛНЕН**

- FastAPI REST сервис (`11_api_server.py`) реализует эндпоинты `POST /index/candidate`, `POST /search/rank` и `POST /score`.
- Pydantic-схемы обеспечивают строгую валидацию входных данных.
- Telemetry HTTP-заголовки: `X-Processing-Time-Ms`, `X-Model-Version`, при этом `X-Model-Version` выводится из basename(`MODEL_PATH`).
- Для запуска микросервиса требуется runtime env `MODEL_PATH`.
- Документация: `docs/obsidian/API_Integration_Final_Report.md` (Раздел 6.5).
