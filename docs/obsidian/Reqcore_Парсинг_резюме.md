# Парсинг резюме в системе Reqcore

**Теги:** #reqcore #parsing #resume #ats #heuristic

В системе Reqcore этап извлечения данных из резюме кандидатов перед отправкой в LLM (языковую модель) строится на классическом (не-нейросетевом) эвристическом анализе.

## Логика работы
1. **Извлечение сырого текста:** В зависимости от типа загруженного файла (проверяется по MIME-типу) используются специализированные библиотеки. За это отвечает модуль `server/utils/resume-parser.ts`:
   - `pdf-parse` для PDF-файлов (обращается к бинарнику и извлекает сырой текст сплошным потоком).
   - `mammoth` для современных документов Word (`.docx`).
   - `word-extractor` для устаревших форматов `.doc`.
2. **Очистка данных (Нормализация):** Полученный текст очищается от лишних пробелов, непечатных символов и множественных переносов строк через встроенную функцию (`normalizeText`), чтобы превратить результат в цельную и чистую "простыню" текста.
3. **Разбивка на секции:** Вместо LLM или сложных NLP-инструментов применяется эвристика на базе строгих регулярных выражений (**RegEx**). В коде задан массив `SECTION_HEADINGS`, который ищет паттерны типа *«Опыт работы»*, *«Education»*, *«Skills»*. 
   - Как только строка совпадает с регулярным выражением (и она короче 60 символов), система помечает её как «Заголовок раздела». 
   - Весь текст, следующий за таким заголовком, записывается в соответствующий блок, вплоть до встречи следующего заголовка.

Такой "очищенный" и разбитый на ключевые блоки текст впоследствии передается контекстом в языковую модель для оценки или генерации вывода.
