## Что нужно доработать

1. Сравнить модели друг с другом, а не только показывать улучшение одной модели после дообучения.
2. Подчеркнуть, что интерфейс системы англоязычный.
3. Добавить денежные метрики: стоимость LLM-скоринга, экономия при top-K pre-ranking, стоимость эксплуатации локальной модели.
4. Сделать сравнение оценок более объективным: не опираться только на baseline `0.3`, а использовать сопоставимые методы и одинаковые условия оценки.
5. Для блока MLOps сделать в Figma отдельный интерфейс/экран мониторинга качества моделей: отслеживание метрик, data drift, model drift и других признаков деградации.
