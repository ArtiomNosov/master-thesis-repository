# Генерация критериев оценки с помощью AI в Reqcore

**Теги:** #reqcore #ai #llm #vercel-ai-sdk #zod #ats #scoring

Модуль Reqcore использует интеграцию с LLM через библиотеку **Vercel AI SDK** для автоматического формирования объективной шкалы оценивания (критериев) кандидатов на основе описания вакансии. Техника опирается на строгий парсинг ответа нейросети в формат JSON.

## Принцип работы
Вместо ручного составления матрицы оценки рекрутером (HR), применяется автоматизированный Pipeline в файле `server/utils/ai/scoring.ts`:
1. **Входные данные:** Система берет текст из поля **Job Description** (описание вакансии) и отправляет его в промпт для LLM (например, OpenAI или Anthropic).
2. **Схемы данных (Zod):** Ключевой элемент архитектуры — использование библиотеки `zod`. Разработчики жестко прописали техническую схему требуемого ответа (`generatedCriteriaSchema`), включающую свойства:
   - `key`: Технический идентификатор (например, `react_skills`).
   - `name`: Название критерия.
   - `description`: Инструкция по поиску фактов в резюме.
   - `category`: Тип (технический, софт-скиллы, образование).
   - `maxScore`: База оценки (всегда 10).
   - `suggestedWeight`: Важность фактора (вес от 10 до 100).
3. **Генерация (generateStructuredOutput):** Во время запроса к LLM используется функция `generateStructuredOutput`. Эта функция заставляет модель отформатировать свой текстовый ответ строго в предоставленную JSON-структуру `Zod`. Модель не может выдать вольный текст, приветствие или лишние комментарии.
4. **Сохранение и использование:** Функция возвращает готовый TypeScript-объект (`result.object.criteria`), который система мгновенно, без дополнительной обработки парсерами строк, сохраняет в базу данных. 

Сгенерированные критерии далее выступают в роли жесткого "чек-листа" для автоматического AI-скоринга входящих резюме (Смотри [[Reqcore_Парсинг_резюме]]).

## Поддерживаемые провайдеры
- **OpenAI**
- **Anthropic**
- **Google AI (Gemini)**
- **OpenAI Compatible Models**
