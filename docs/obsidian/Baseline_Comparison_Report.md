## 1. Конфигурация бенчмарка
В сравнении должны участвовать ровно три модели, отражающие три принципиально разные стратегии ранжирования кандидатов:

1. **BM25 lexical ranking baseline** — классическая модель информационного поиска без обучения. Токенизация выполняется простой регулярной токенизацией по словам с приведением к нижнему регистру. Score рассчитывается как BM25 relevance score между текстом вакансии как query и текстом резюме как document.
2. **Fine-tuned Cross-Encoder** — `cointegrated/rubert-tiny2` в архитектуре sequence classification для бинарной релевантности пары. Модель обучается на `train.tsv`, валидируется на `val.tsv`, финально оценивается только на `test.tsv`. Вход: `[vacancy_text, resume_text]`; output score: вероятность класса `label = 1`.
3. **Fine-tuned Bi-Encoder** — основная модель проекта `experiments/models/bi_encoder_rubert_tiny2`. Вакансия и резюме кодируются независимо, а score задается cosine similarity между их embedding. Эта архитектура проверяется как proposed approach, пригодный для массового ранжирования в ATS.

Скрипт `experiments/scripts/12_baseline_comparison.py` приведен к этой схеме: прежние черновые baseline-варианты больше не входят в основной сравнительный протокол. Он поддерживает демонстрационный edge-case и режим `--evaluate_split` для финальной оценки на `test.tsv`. Для обучения cross-encoder добавлен отдельный воспроизводимый контур `experiments/scripts/18_train_cross_encoder.py`.

## 2. Исследовательский смысл сравнения

### Подход 1: BM25 lexical ranking baseline
BM25 проверяет, насколько далеко можно продвинуться только за счет лексического совпадения терминов, навыков и должностей. Он важен как прозрачная и дешевая нижняя граница качества: модель не требует обучения, не использует нейросетевой inference и хорошо интерпретируется, но плохо переносит синонимию, перефразирование и различия в стиле описания опыта.

### Подход 2: Fine-tuned Cross-Encoder
Cross-encoder проверяет верхнюю границу качества для дорогого pairwise scoring. Благодаря совместной обработке пары через attention модель может учитывать тонкие соответствия между требованиями вакансии и опытом кандидата, однако каждый кандидат требует отдельного forward pass вместе с текстом вакансии. Поэтому такой вариант выразителен, но хуже подходит для массового ранжирования больших пулов без предварительного отбора.

### Подход 3: Fine-tuned Bi-Encoder
Bi-encoder проверяет практически применимую embedding-архитектуру: резюме можно закодировать заранее, хранить embeddings в индексе и при появлении вакансии считать только embedding запроса и cosine similarity. Это снижает стоимость inference и делает подход естественным для интеграции в ATS, где требуется быстро сортировать большие списки кандидатов.

## Практические заключения для диссертации
Главный вывод эксперимента должен формулироваться не как механическое сравнение метрик, а как анализ компромисса между качеством, скоростью, стоимостью inference и пригодностью для ATS. BM25 задает классическую лексическую базовую линию; cross-encoder показывает потенциальный максимум качества при дорогом попарном сопоставлении; bi-encoder оценивает масштабируемый семантический вариант, который можно использовать как основной retrieval/ranking слой в промышленном контуре.
