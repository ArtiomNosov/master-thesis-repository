# Контекст и План Презентации Магистерской Диссертации

## Полный контекст диссертации

**Тема:** Разработка и внедрение интеллектуального модуля ранжирования кандидатов на основе семантического анализа резюме и вакансий в систему автоматизированного подбора персонала.

**Проблематика:** 
Современный рекрутмент сталкивается с высокой нагрузкой на этапе первичного отбора. В традиционных ATS-системах (системы управления кандидатами) ранжирование и фильтрация больших пулов соискателей — это ресурсоемкий процесс. Классический BM25-поиск по навыкам работает для прямых совпадений, но не улавливает семантических связей в тексте, тогда как использование продвинутого LLM-скоринга для абсолютно всех кандидатов — экономически и вычислительно дорого.

**Цель работы:**
Спроектировать и реализовать модуль сопоставления «вакансия–резюме» на базе плотных семантических векторных представлений текста (эмбеддингов) и интегрировать его в ATS-систему (Reqcore) для эффективного предварительного отбора кандидатов. 

**Ключевые решения и реализация:**
1. **Алгоритмический фундамент:** Использование сиамских нейронных сетей (Sentence-BERT / bi-encoder). Выбрана базовая модель `cointegrated/rubert-tiny2`.
2. **Обучение модели:** Был собран и подготовлен корпус данных из агрегатора вакансий и резюме. Модель была дообучена (fine-tuning) с функцией потерь ContrastiveLoss на парах «вакансия-резюме», извлеченных из исторических HR-событий (приглашения/отказы).
3. **Результаты оценки (Метрики):** 
   - Основной эксперимент сравнивает ровно три стратегии: BM25 lexical baseline, fine-tuned cross-encoder и fine-tuned bi-encoder.
   - Для proposed bi-encoder на задаче бинарного сопоставления достигнуты Average Precision 0.9870 на `val.tsv` и 0.9921 на `test.tsv`.
   - На задаче ранжирования (по тестовому срезу вакансий): достигнуто значение NDCG@3 = 1.0, Precision@3 = 0.9958.
4. **Интеграционный слой:** Написан Python API Service (FastAPI) с маршрутами индексации (`/index/candidate`) и ранжирования (`/search/rank`).
5. **Интеграция с ATS (Reqcore):** Модель работает как предварительный шаг. Решение внедрено как ветка скоринга `provider === 'biencoder'`. Bi-encoder значительно удешевляет скоринг: вместо дорогого вызова LLM (например, OpenAI gpt-4o-mini) для сотен кандидатов, локальная ML-модель рассчитывает cosine_similarity и делает быстрый пред-отбор (top-K), что снижает затраты на внешний API (по оценке до 90%).
