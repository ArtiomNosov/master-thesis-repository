# Контекст и План Презентации Магистерской Диссертации

## Полный контекст диссертации

**Тема:** Разработка и внедрение интеллектуального модуля ранжирования кандидатов на основе семантического анализа резюме и вакансий в систему автоматизированного подбора персонала.

**Проблематика:** 
Современный рекрутмент сталкивается с высокой нагрузкой на этапе первичного отбора. В традиционных ATS-системах (системы управления кандидатами) ранжирование и фильтрация больших пулов соискателей — это ресурсоемкий процесс. Классический поиск по навыкам (TF-IDF, BM25) работает, но не улавливает семантических связей в тексте, тогда как использование продвинутого LLM-скоринга для абсолютно всех кандидатов — экономически и вычислительно дорого. 

**Цель работы:**
Спроектировать и реализовать модуль сопоставления «вакансия–резюме» на базе плотных семантических векторных представлений текста (эмбеддингов) и интегрировать его в ATS-систему (Reqcore) для эффективного предварительного отбора кандидатов. 

**Ключевые решения и реализация:**
1. **Алгоритмический фундамент:** Использование сиамских нейронных сетей (Sentence-BERT / bi-encoder). Выбрана базовая модель `cointegrated/rubert-tiny2`.
2. **Обучение модели:** Был собран и подготовлен корпус данных из одного агрегатора вакансий и резюме (каталог данных агрегатора). Модель была дообучена (fine-tuning) с функцией потерь ContrastiveLoss на парах «вакансия-резюме», извлеченных из исторических HR-событий (приглашения/отказы).
3. **Результаты оценки (Метрики):** 
   - На задаче бинарного сопоставления: Average Precision вырос с 0.2590 у базовой `cointegrated/rubert-tiny2` без дообучения до 0.9870 после fine-tuning; F1 вырос до 0.9605. Это сравнение показывает вклад доменного дообучения, но остается офлайн-оценкой на подготовленном split.
   - На задаче ранжирования (по тестовому срезу вакансий): достигнуто значение NDCG@3 = 1.0, Precision@3 = 0.9958.
4. **Интеграционный слой:** Написан Python API Service (FastAPI) с маршрутами индексации (`/index/candidate`) и ранжирования (`/search/rank`).
5. **Интеграция с ATS (Reqcore):** Модель работает как предварительный шаг. Решение внедрено как ветка скоринга `provider === 'biencoder'`. Bi-encoder может снижать число внешних LLM-вызовов за счет локального cosine_similarity и предварительного top-K отбора; оценка экономии зависит от выбранных N и K и является расчетным сценарием, а не production billing export.
