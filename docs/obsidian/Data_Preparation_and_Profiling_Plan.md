## 1. Исходные данные (агрегатор вакансий Extract)
В директории `каталога данных агрегатора/` был обнаружен набор сырых данных:
- **Резюме** (`dwh_dwh_region_resume1_ods.tsv`): ~17.3 ГБ. Содержит поля профиля кандидата (ФИО, контакты, зарплатные ожидания, желаемая профессия и т.д.). Требуется агрессивная деидентификация (удаление ФИО, телефонов, email).
- **Вакансии** (`dwh.hr_vacancy_ods_and_ods_hr_storage_vacancy.tsv`): ~44.9 ГБ. Наиболее объемный файл, содержащий текстовые описания, требования, условия и метаданные вакансий.
- **События откликов/взаимодействий** (`dwh_dwh_hr_received_resume_event_ods.tsv`): ~293 МБ. Является ключевым файлом для формирования связей между кандидатами и вакансиями (positive pairs) для задач мэтчинга и ранжирования.
- **Дополнительные таблицы**: Опыт работы, навыки (`professional_skill`), образование, социальные сети и т.д.

## 2. Проблематика и особенности процессинга
- Ввиду большого размера файлов (десятки гигабайт) анализ и предобработку с использованием Pandas необходимо будет выполнять чанками (chunking), либо перевести пайплайн на Dask / Polars.
- Текстовые поля (опыт работы, обязанности, описания) могут содержать HTML-теги, грязные символы и стоп-слова, которые нужно предварительно очистить.

## 3. Запланированные шаги (согласно разделу 6.1 ТЗ)
1. **Сэмплирование:** Выделение репрезентативной выборки (например, 10-50 тыс. записей) для быстрой разработки и отладки алгоритмов чистки.
2. **Очистка и деидентификация:** Разработка скрипта нормализации форматов, очистки текстов (шум, дубликаты) и маскирования ПДн (ФИО, телефоны, email).
3. **Профилирование:** Подсчет статистик:
   - Объем датасета и средняя длина документов.
   - Распределения по доменам, языкам (важно для энкодеров), seniority (уровню) и скиллам.
4. **Генерация меток (Labels) и разбиение:** Использование таблицы `received_resume_event` для формирования позитивных пар (кандидат откликнулся на вакансию) и генерация негативных пар (hard negatives / random negatives).
5. **Сплитование:** Создание `train / validation / test` выборок для последующего обучения Semantic matching модели.
